# Game Development & Distribution Guide

**For Lumorix developers who want to create games independently, package them, and publish to GitHub Releases.**

## Architecture Overview

```
Your Game (GitHub Repo)
├── src/
│   ├── index.html        ← Launcher starts here
│   ├── main.ts           ← Your game logic
│   └── style.css
├── .github/workflows/
│   └── release.yml       ← Automated release pipeline
└── package.json

                  ↓ npm run package ↓

game.zip                 ← Packaged game (all src compiled)
+ game.zip.sha256        ← Integrity check
+ manifest.json          ← Metadata for Launcher

                  ↓ git tag v1.0.0 ↓

GitHub Release
├── game.zip
├── game.zip.sha256
└── manifest.json

                  ↓ User adds to Launcher ↓

Launcher reads manifest.json
→ Downloads & verifies game.zip
→ Extracts to user library
→ Player clicks "Play"
→ Launcher opens index.html
```

## Quickstart: Create Your First Game

### 1. Use the Game Template

The template is located at `game-template/` in this repository.

```powershell
# Clone or copy the template
cp -r game-template my-awesome-game
cd my-awesome-game

# Install dependencies
npm install

# Start developing
npm run dev
# → Open http://localhost:3000
```

### 2. Edit Your Game

```typescript
// src/main.ts - Your game runs here
console.log('🎮 My Game Started!');

// Example: Handle a button click
document.getElementById('start-btn')?.addEventListener('click', () => {
  console.log('Player pressed START');
  // Your game logic...
});
```

### 3. Build & Package Locally

```powershell
# Full build + ZIP + SHA256
npm run package

# Outputs:
# - dist/             (Web app build)
# - game.zip          (Ready for upload)
# - game.zip.sha256   (Checksum for verification)
```

### 4. Test in Launcher (Optional)

Copy the unpacked `dist/` folder path and add it as a local manifest to test before uploading.

### 5. Push to GitHub & Release

```powershell
# Commit your game
git add .
git commit -m "v1.0.0: My Awesome Game Release"
git tag v1.0.0
git push origin main --tags

# GitHub Action runs automatically:
# 1. Builds your game
# 2. Creates game.zip + SHA256
# 3. Generates manifest.json
# 4. Pushes Release to GitHub
```

### 6. Add to Launcher

The manifest is now available at:
```
https://github.com/username/my-awesome-game/releases/download/v1.0.0/manifest.json
```

**Option A: Central Registry** (future)
```
POST to Lumorix game-catalog with manifest URL
```

**Option B: Manual in Launcher**
```
1. Download manifest.json from release
2. Save to %LOCALAPPDATA%\Lumorix Launcher\Manifests\
3. Restart Launcher
```

**Option C: Configure Manifest Source in Launcher**
```json
{
  "manifestSources": [{
    "url": "https://github.com/username/my-awesome-game/releases/latest/download/manifest.json",
    "type": "customHttp",
    "enabled": true
  }]
}
```

## Complete Example: "Click Counter" Game

### Step 1: Create files

**src/index.html**
```html
<!DOCTYPE html>
<html>
  <head>
    <title>Click Counter</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <div class="game-container">
      <h1>Click Counter</h1>
      <div id="counter" class="counter">0</div>
      <button id="click-btn" class="btn">CLICK ME!</button>
      <button id="reset-btn" class="btn btn-secondary">Reset</button>
    </div>
    <script type="module" src="main.ts"></script>
  </body>
</html>
```

**src/style.css**
```css
body {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  font-family: Arial, sans-serif;
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
  margin: 0;
}

.game-container {
  text-align: center;
  background: white;
  padding: 3rem;
  border-radius: 20px;
  box-shadow: 0 20px 60px rgba(0,0,0,0.3);
}

h1 { color: #333; font-size: 2.5rem; }

.counter {
  font-size: 4rem;
  color: #667eea;
  font-weight: bold;
  margin: 2rem 0;
}

.btn {
  padding: 1rem 2rem;
  font-size: 1.2rem;
  background: #667eea;
  color: white;
  border: none;
  border-radius: 10px;
  cursor: pointer;
  margin: 0.5rem;
  transition: all 0.3s;
}

.btn:hover { background: #764ba2; }
.btn-secondary { background: #999; }
```

**src/main.ts**
```typescript
let count = 0;

const counterEl = document.getElementById('counter')!;
const clickBtn = document.getElementById('click-btn')!;
const resetBtn = document.getElementById('reset-btn')!;

function updateDisplay() {
  counterEl.textContent = String(count);
}

clickBtn.addEventListener('click', () => {
  count++;
  updateDisplay();
});

resetBtn.addEventListener('click', () => {
  count = 0;
  updateDisplay();
});
```

### Step 2: Create manifest

**manifest.json** (auto-generated by GitHub Action)
```json
{
  "id": "com.lumorix.click-counter",
  "name": "Click Counter",
  "version": "1.0.0",
  "description": "A simple clicking game to test the Lumorix Launcher",
  "developer": "Your Name",
  "releaseDate": "2026-04-24",
  "tags": ["Casual", "Idle"],
  "executable": "index.html",
  "installSizeBytes": 15000,
  "defaultInstallFolder": "Click Counter",
  "supportedActions": ["install", "launch", "uninstall", "openFolder"],
  "installStrategy": { "kind": "zipArchive", "rootFolder": "." },
  "download": {
    "kind": "httpArchive",
    "url": "https://github.com/yourname/click-counter/releases/download/v1.0.0/game.zip",
    "sha256": "abc123...",
    "sizeBytes": 15000
  },
  "changelog": [{
    "version": "1.0.0",
    "date": "2026-04-24",
    "items": ["Initial release"]
  }]
}
```

### Step 3: Release

```powershell
git tag v1.0.0
git push origin main --tags

# Wait for GitHub Action to complete...
# Release available at:
# https://github.com/yourname/click-counter/releases/v1.0.0
```

## Key Concepts

### 1. Web-based Games
Your game is a **web application** (HTML/CSS/JS/TS) running in a browser window. No special skills needed beyond web development.

### 2. Sandboxed Execution
- No direct filesystem access
- No registry access
- No system command execution
- → Safer for players

### 3. Automatic Updates
Update your game by:
```powershell
# Increment version in package.json
npm run package
git tag v1.1.0
git push --tags
# → GitHub Action rebuilds & releases
# → Launcher detects update
# → Player can click "Update"
```

### 4. Asset Management
All assets (images, sounds, fonts) must be in the ZIP:
```
game.zip/
├── index.html
├── assets/
│   ├── sprites/
│   ├── sounds/
│   └── fonts/
└── ...
```

Reference them with **relative paths**:
```html
<img src="./assets/sprites/player.png" />
<audio src="./assets/sounds/jump.mp3"></audio>
```

## Security & Integrity

1. **SHA256 Verification**: Launcher verifies game.zip matches the checksum
2. **Manifest Signing** (future): Manifests can be cryptographically signed
3. **Sandboxing**: Games run in isolated browser context

## Troubleshooting

### Game won't start
- [ ] `index.html` exists in game.zip root
- [ ] `main.ts` compiles without errors
- [ ] All asset paths are relative (./assets/...)
- [ ] Check browser console in launcher (F12)

### Launcher won't recognize game
- [ ] Manifest JSON is valid
- [ ] SHA256 matches actual file
- [ ] `id` is unique
- [ ] `executable` matches entry point

### Large file size
- [ ] Remove `node_modules` from ZIP (shouldn't be there)
- [ ] Optimize images (Tinypng, Squoosh)
- [ ] Minify CSS/JS (default in Vite)
