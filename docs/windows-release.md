# Windows Release and Signing Checklist

This checklist documents the production setup for signed Lumorix launcher releases.

## 1) Rust and MSVC Toolchain

- Install `rustup` and select `stable-x86_64-pc-windows-msvc`.
- Install Visual Studio Build Tools 2022 with `Desktop development with C++`.
- Ensure `signtool.exe` is available from Windows SDK.

## 2) Tauri Build Configuration

Configured in `src-tauri/tauri.conf.json`:

- `bundle.createUpdaterArtifacts: true`
- `bundle.windows.digestAlgorithm: "sha256"`
- `bundle.windows.timestampUrl: "http://timestamp.digicert.com"`
- `bundle.windows.signCommand: "powershell ... scripts/sign-windows.ps1 %1"`
- `plugins.updater.endpoints`: points to `latest.json` in GitHub Releases
- `plugins.updater.pubkey`: updater public key generated by `tauri signer generate`
- `src-tauri/capabilities/default.json`: includes `updater:default` for in-app check, download and install

Release version and latest changelog metadata now come from `release-info.json` in the project root. The sync step updates `src-tauri/Cargo.toml`, `src-tauri/tauri.conf.json`, `package.json`, `package-lock.json`, and `distribution/latest.json`.

## 3) Code Signing Secrets

Required CI secrets/environment variables:

- `TAURI_SIGNING_PRIVATE_KEY`: updater private key content
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: optional updater key password
- `LUMORIX_SIGN_CERT_SHA1`: code signing certificate thumbprint
- `LUMORIX_SIGN_TIMESTAMP_URL`: optional RFC3161 timestamp endpoint

The signing script is `src-tauri/scripts/sign-windows.ps1`.

## 4) Launcher Updater Release Artifacts

Each launcher release must publish:

- Installer (`.msi` or `.exe`)
- Updater archive generated by Tauri (zip)
- Signature file generated by Tauri updater process
- `latest.json` with per-platform URLs and signatures

Reference example: `distribution/launcher-latest.example.json`.

Automated publishing is implemented in `.github/workflows/release-windows.yml`.

`distribution/latest.json` is generated from `release-info.json` before the build and then finalized in CI with the real updater asset URL, signature, and GitHub release URL.

The in-app update buttons in About and Settings call Tauri's updater plugin. The plugin checks the configured endpoint, verifies the signed artifact with the public key in `tauri.conf.json`, downloads it and starts installation. On Windows, Tauri exits the app during the install step.

## 5) Game Package Release Artifacts

Per game release in GitHub Releases:

- Signed game zip asset
- Detached signature file (`.sig`)
- SHA-256 manifest text file
- Updated game manifest (`distribution/manifests/games/*.json`) with final `download.sha256` and `sizeBytes`

## 6) Final Installer Metadata

Before final production release, validate:

- Product name and identifier in `src-tauri/tauri.conf.json`
- Release version, title, date, and notes in `release-info.json`
- Bundle descriptions (`shortDescription`, `longDescription`)
- Icons in `src-tauri/icons/`
- Legal metadata in installer branding assets and signing certificate subject
